# 加密备份与恢复

## 用途与边界

`.wootodo` 是用户主动导出的完整加密快照，用于设备损坏、系统重装或同步服务故障后的恢复。它不是实时同步文件，不能让 macOS 与 Android 同时覆盖同一份网盘文件。

Android 端明确禁用系统云备份与设备迁移备份，避免任务数据库绕过 `.wootodo` 加密流程；换机或重装时应使用本功能手动恢复。

备份解密正文包含：

- 全部待办、完成和 Pass 历史；
- 任务 ID、周期、任务线、排序与原始结算时间；
- 已删除实体的 tombstone 屏障，避免重复任务实例在恢复后按确定性 ID 复活；
- 可选的已绑定设备 endpoint、vault ID、device ID、设备令牌和 vault key；
- 协议版本与导出时间。

服务端不会收到备份口令或备份文件。文件外层只暴露格式版本、导出时间和密钥派生参数，不暴露任务标题或同步身份。

## 加密格式

- 口令先做 Unicode NFKC 规范化，再使用 UTF-8 字节。
- PBKDF2-HMAC-SHA256，默认 210000 轮，随机 16 字节 salt。
- AES-256-GCM，随机 12 字节 nonce，16 字节认证标签。
- AAD：`woo-todo-backup-v1|createdAt|pbkdf2-hmac-sha256|iterations|salt|aes-256-gcm`。
- 二进制字段使用无填充 Base64URL。
- 任务与 tombstone 合计最多 50000 条、32 MiB 密文；跨数组重复实体 ID、重复 JSON 键、未知字段和认证失败均拒绝导入。

`v0.1.1` 将 `tombstones` 作为 v1 正文的可选字段：旧版备份缺失该字段时按空数组读取，现有 golden vector 保持不变；包含删除屏障的新备份需要使用 `v0.1.1` 或更高版本恢复，旧客户端会按严格 JSON 规则拒绝它，而不会静默忽略删除语义。

跨端 Schema 与 golden vector 位于 `shared/schema/backup*.json` 和 `shared/fixtures/backup-vectors.json`。

## 导出

1. 在应用的同步/设置区域选择“导出 `.wootodo`”。
2. 输入至少 10 个 Unicode code point 的独立口令，并再次确认。
3. 默认不要勾选“包含同步身份”。只有原安装即将被替换、且不会与恢复后的副本同时运行时才开启。
4. 保存为带日期的文件，例如 `WooTodo-2026-07-15.wootodo`。
5. 将文件手动上传到夸克网盘。
6. 在密码管理器中单独保存口令，不要把口令写在同一个网盘目录。

建议保留最近两到三个版本。导出新备份后，不要立刻删除上一个可用版本。

## 全新安装恢复

1. 安装并打开 Woo Todo，暂时不要创建任务或连接新的同步空间。
2. 选择“从 `.wootodo` 恢复”，从系统文件选择器读取下载好的文件。
3. 输入原备份口令。
4. 应用先验证文件大小、严格 JSON、PBKDF2/AES-GCM 认证、全部任务字段与删除屏障，再写入本地 SQLite。
5. 若文件包含同步恢复材料，应用恢复系统安全存储并触发同步；若不包含，则恢复任务后按正常二维码流程把它作为新设备配对。同步服务不可用时，本地任务仍可使用。

导入只允许任务库为空且没有现有同步身份，避免把两个历史空间静默混合。正常增加第二台设备必须使用二维码配对。

备份会恢复原来的逻辑设备 ID 与令牌，因此它适合“替换原设备或重装”场景，不应同时导入到多台仍活跃的设备。旧设备仍可使用时，恢复后应在设备列表撤销旧设备，或直接使用二维码添加新设备。

## 无法恢复的情况

- 口令遗失：AES-GCM 密文无法解密。
- 文件损坏或被篡改：认证失败，应用不会写入部分任务。
- 全部设备、备份文件和口令同时丢失：服务端只有任务密文，无法代为找回。

夸克网盘只负责保存已加密文件。即使网盘账号泄露，攻击者仍需猜中备份口令；因此不要使用过短或与网盘相同的密码。
