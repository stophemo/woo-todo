# 坚果云自动同步与通知

这条路径适合两台设备都能访问坚果云、但不想购买服务器或配置代理的个人使用。请先安装并登录坚果云官方客户端，确认同一账号在两端可用；Woo Todo 不通过坚果云网页上传文件，而是由内置 WebDAV 客户端直接连接坚果云。应用不需要 Woo Todo 账号，也不需要部署 Worker；任务仍然先写入本地数据库，网络只负责把加密变更送到另一端。

从 `v0.1.5` 起，Android 可在 Woo Todo 内直接扫描 Mac 配置二维码；当前从首页未连接状态点击“配对”后选择“扫描二维码配对”，也可以选择“手动填写坚果云配置”。系统相机/扫码器打开 Woo Todo 仍作为备用入口。不同入口使用相同的 WebDAV 配置和安全边界。

## 1. 准备坚果云应用密码

在坚果云网页打开“账户信息 → 安全选项 → 第三方应用管理”，创建一个应用密码。网页端这里只用于账户安全设置和生成应用密码，不提供 Woo Todo 所需的 WebDAV 同步。不要把网页登录密码填入 Woo Todo。两台设备使用同一个坚果云账号。配置二维码默认会把 Mac 当前应用密码带到 Android；若希望以后能单独撤销某台设备，可先为 Android 创建第二个应用密码，扫码预填后替换密码再保存。

## 2. 先在 Mac 配置首台设备

1. 在 macOS 打开菜单栏“设置… → 同步 → 坚果云自动同步”。
2. 填写坚果云账号邮箱和应用密码；首次配置时让应用生成同步空间名和同步密钥。同步空间名是一段短文本，同步密钥是 Base64URL 字符串。
3. 点击“保存并连接”，等待显示“坚果云已连接”。首次绑定会把 Mac 本机已有任务写成加密操作对象。
4. 配置成功后，二维码默认保持隐藏；需要配置 Android 时，显式点击“显示 Android 配置二维码”。二维码和“复制完整配置链接”只用于把同一组配置带入 Android，不是自建 Worker 的配对二维码。

## 3. Android 扫码带入配置

1. 让 Mac 与 Android 保持近旁，在 Mac 上点击“显示 Android 配置二维码”。
2. 在 Android 打开 Woo Todo，首页未连接时点击“配对 → 扫描二维码配对”，允许相机权限后把 Mac 二维码放入取景框。内置扫码器只接受 Woo Todo 的坚果云配置或 Worker 配对链接，不会把结果交给其他 App。
3. Woo Todo 会严格解析 `username`、`appPassword`、`vaultId` 和 `vaultKey`，预填确认页；二维码不包含固定 WebDAV 端点或 `deviceId`。确认来源和预填内容后点击保存，Android 会生成独立 `deviceId` 并立即执行首次同步。
4. 配置二维码和复制出的深链同时包含坚果云 WebDAV 应用密码与 Woo Todo 同步密钥，等同完整敏感凭据；拿到它的人可能访问该坚果云账号允许 WebDAV 访问的文件，并解密 Woo Todo 同步对象。只在两台设备旁展示；用完立即在 Mac 隐藏二维码，并覆盖可能留在系统剪贴板中的配置链接。绝不发到群聊、论坛、公开仓库、截图或日志；Woo Todo 不会上传或记录二维码和完整配置链接。
5. 首次同步失败不会撤销已保存配置：网络类失败会自动重试，认证或格式错误会在首页同步状态中提示。内置扫码不可用时，可用系统相机打开 Woo Todo，或在“配对 → 手动填写坚果云配置”手动填写；使用系统扫码器时必须确认实际打开的是 Woo Todo，因为 `wootodo://` 自定义 scheme 不能证明应用归属。

同步密钥一旦使用，不要在任一端重新生成或改写；需要换空间时先导出加密恢复备份。云端可看到对象路径、操作/实体/设备 ID、变更类型、逻辑时钟等同步元数据和密文，但看不到任务标题；丢失两端密钥与备份后无法恢复任务正文。

## 4. 自动同步何时发生

- 本地新增、编辑、完成、Pass、删除或排序后立即写入 outbox，并排队同步；网络不可用时任务仍可正常使用。
- Android 使用系统联网 Job，通常不短于 15 分钟一次，系统省电策略可能延后；macOS 在启动、本地变更和约 15 分钟兜底计时器触发。
- “立即同步”只是在当前网络条件下提前执行一次，不会绕过坚果云权限或系统网络限制。
- WebDAV 历史对象不会自动删除，tombstone 会长期保留以防删除任务被旧设备复活。

如果两端都没有成功连接，请先在浏览器确认坚果云账号可用，再检查是否误填了网页登录密码、空间名/密钥是否有空格，以及系统是否允许应用访问网络。应用不会因为同步失败而阻塞任务编辑。

## 5. 设置每条待办的通知

在新增或编辑任务时打开“在指定时间提醒”，选择当天的时间并保存。只有有日期的待办支持任务提醒，闲时任务、已完成任务和 Pass 任务不会排程。提醒时间随任务一起进入加密同步对象，另一端收到后会用自己的系统通知机制重新排程，不会把通知权限或已显示状态同步到云端。

- macOS：首次有待办提醒时系统会请求通知权限；拒绝后可在“系统设置 → 通知 → Woo Todo”开启。
- Android：首次启动会请求通知权限；拒绝后可在“系统设置 → 应用 → Woo Todo → 通知”开启。Android 的省电和系统闹钟策略可能使通知略有延迟。

通知是双端本地行为，不依赖坚果云在线。任务在一端完成或删除后，同步到另一端会取消对应的本地提醒。

## 6. 两种在线方式不能同时启用

同一个本地任务库只能绑定坚果云或自建 Worker 其中一种在线同步方式。若已配置 Worker，请先按现有同步页停用或恢复空白本地库，再配置坚果云；不要把两个空间名或两套密钥混用。
