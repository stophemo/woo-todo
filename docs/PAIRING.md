# macOS 与 Android 配对同步

Woo Todo 没有传统账号。Mac 先创建一个加密同步空间，再通过 10 分钟有效的二维码把 Android 安全加入同一空间。两端始终先写本地 SQLite；同步服务只保存端到端加密后的数据。

## 先准备同步服务地址

创建空间时填写的必须是 Mac 与 Android 都能访问的 Woo Todo Cloudflare Worker HTTPS 根地址，例如：

```text
https://woo-todo-sync.<你的子域>.workers.dev
```

- Vercel 地址是产品主页，不是同步后端。
- 夸克网盘可以保存 `.wootodo` 加密备份，但不能充当实时同步后端。
- 地址末尾不要添加 `/v1`，应用会自动拼接 API 路径。
- 还需向同步服务部署者取得“创建空间邀请码”；它只用于首次创建，不会保存在客户端，也不是稍后两端核对的六位设备配对码。
- 如果还没有 Worker，请先按 [后端部署说明](../backend/README.md) 创建 D1、配置 `TOKEN_PEPPER` 与 `VAULT_CREATION_INVITE_CODE`、执行远端迁移并部署 Worker。

### 为什么不能填写 `http://127.0.0.1`

`127.0.0.1` 永远表示“当前这台设备”，不是某一台固定电脑：

- Mac 访问 `127.0.0.1` 时，连接的是 Mac 自己；
- Android 访问同一个地址时，连接的是手机自己；
- `http://127.0.0.1` 没写端口时默认访问 80 端口，而 `wrangler dev` 通常监听 `http://127.0.0.1:8787`。

所以 `127.0.0.1:8787` 只适合在 Mac 上调试 Worker，不能放进真实双机配对二维码。真实配对必须使用双方可访问、证书有效的 HTTPS 地址。

## 第一次创建空间

1. 在 Mac 菜单栏点击 Woo Todo 图标，选择“任务详情与统计”。
2. 在窗口左侧选择“同步”。
3. 填写已部署的 HTTPS Worker 根地址和部署者提供的创建空间邀请码；自托管时填写你为 `VAULT_CREATION_INVITE_CODE` 设置的值。
4. 确认界面显示“地址格式正确”，再点击“创建同步空间”。邀请码只随这一次 HTTPS 请求发送，应用不会保存。
5. 创建成功后，Mac 会显示服务地址、同步空间和当前设备。Android 不需要再次输入这些内容。

如果提示无法连接，请先在 Mac 和手机浏览器分别打开：

```text
https://你的-worker.workers.dev/health
```

两端都应能获得 Worker 的健康检查响应。能打开 Vercel 主页并不能证明同步 Worker 已部署。

## 把三星手机加入空间

当前 Android 版没有应用内扫码器，实际使用三星系统相机或系统二维码扫描器唤起 Woo Todo：

1. 在 Mac 的“同步”页找到“添加 Android 设备”，点击“生成配对二维码”。
2. 在 Galaxy S23 Ultra 下拉快捷面板中打开“扫描二维码”；也可以使用系统相机的二维码识别。
3. 扫描 Mac 上的二维码，在扫码结果中点按“打开 Woo Todo”。
4. 手机开始建立安全连接，两端随后会显示同一个六位核对码。
5. 逐位确认两端数字完全一致。
6. 回到 Mac，点击“核对一致，确认绑定”。不要在核对码不同的情况下确认。
7. 手机保存同步密钥后会自动执行首次同步。回到 Android 首页，看到“同步完成”即表示连接成功。

如果系统扫码器没有提供“打开 Woo Todo”，可以在 Mac 点击“复制配对链接（备用）”，通过只属于自己的私密渠道发到手机后点按链接。链接包含 10 分钟有效的一次性 secret，不要发到群聊、论坛或公开仓库。

## 验证双向同步

1. 在 Mac 新建一个容易识别的测试任务，点击同步页的“立即同步”。
2. 在 Android 首页点击“同步”，确认任务出现。
3. 在 Android 新建另一个测试任务并再次同步。
4. 回到 Mac 点击“立即同步”，确认手机任务出现。

同步失败不会阻止任一端继续编辑本地任务，网络恢复后可以重试。

## 常见错误

| 提示或现象 | 原因 | 处理方式 |
| --- | --- | --- |
| `127.0.0.1/localhost 只指向当前设备` | 填写了本机调试地址 | 部署 Worker，改用两端可访问的 HTTPS 地址 |
| 无法连接同步服务 | Worker 未部署、地址错误、网络不可达 | 检查 `/health`、Worker 部署状态和两端网络 |
| 找不到创建空间接口 | 填了 Vercel 页面、其他网页，或手动追加了 `/v1` | 改填 Worker 根地址 |
| 创建空间邀请码无效 | 邀请码缺失、输错，或与 Worker secret 不一致 | 向部署者确认；自托管时重新配置 `VAULT_CREATION_INVITE_CODE` |
| 找不到同步服务 | 域名无法解析或 Worker 域名不可达 | 在手机浏览器检查 `/health`，必要时更换可达域名 |
| HTTPS 证书校验失败 | 使用了自签名或无效证书 | 使用 Cloudflare 提供的有效 HTTPS 域名 |
| 二维码已过期 | 配对超过 10 分钟 | 在 Mac 重新生成二维码 |
| 六位码不同 | 扫描对象或临时密钥不一致 | 立即取消，不要确认；重新生成并扫描 |
| 本机已完成配对 | Android 已保存另一份同步身份 | 不要反复扫码，先按下方说明处理现有绑定 |

## 撤销设备与重新配对

Mac 的“已绑定设备”列表可以撤销其他设备。撤销后：

- 被撤销设备不能再访问同步服务；
- 它已经下载的本地任务不会被远程删除；
- 撤销不等同于清除该设备的本地同步身份。

当前版本尚未提供安全的自助“解除本机绑定”按钮。同步身份同时存在于 Keychain/Android Keystore 和 SQLite 同步元数据中，只删除凭据会留下无法重新配对的半绑定状态，因此不要手动删除 Keychain 项或应用内部凭据文件。

Android 必须重新安装时，可使用以下保守流程：

1. 如果旧连接仍可用，先在 Android 手动同步，并确认 Mac 已收到最新任务。
2. 在 Android 导出加密备份；如目的是创建全新设备身份，请关闭“包含同步身份”。
3. 在 Mac 的设备列表撤销旧 Android 设备。
4. 卸载并重新安装 Android 应用。
5. 如有仅保存在手机的任务，先向空白安装导入不含同步身份的备份。
6. 在 Mac 重新生成二维码并按正常流程配对。

如果旧 Android 已无法同步且含有独有的未同步修改，请先保留加密备份，不要直接清除数据。Mac 本机需要更换同步空间或本机身份损坏时，当前版本也不应通过只删 Keychain 强行重置；需要后续提供事务化清理同步元数据的正式入口。

## 配对安全边界

- 二维码中的配对 secret 和临时公钥 10 分钟后失效。
- 两端使用 X25519 建立临时会话，再用六位码人工确认没有扫错对象。
- vault key 只在确认后加密传给新设备，服务端不会获得任务明文。
- 六位码只能用于人工核对，不能替代二维码中的随机 secret。
