# 无我待办 v1 产品规格

## 1. 核心场景

用户在晚上睡前通过 Android 手机列好明日任务；第二天到公司唤醒 Mac 后，透明悬浮任务板直接显示今日清单。任务板保持可见但不妨碍工作，用户可通过全局快捷键切换交互与鼠标穿透状态。

首版用户是项目维护者本人，目标设备为：

- MacBook Air，Apple M4，macOS Tahoe 26.5.2
- Samsung Galaxy S23 Ultra，Android 16，One UI 8.0

项目公开开源，但首版只承诺上述两台真机的完整验收。

## 2. 任务模型

### 2.1 两个独立维度

每条任务同时具有一个时间类型和一个任务级别。

时间类型：

- `day`：分配到某一天
- `week`：分配到某一自然周，周一开始
- `month`：分配到某一自然月
- `someday`：没有结束时间，闲时完成

任务级别：

- `main`：主线
- `side`：支线
- `extra`：外传

默认按主线、支线、外传分组展示；组内由用户手动排序。

### 2.2 一次性与重复

- 一次性任务直接创建一个任务实例。
- 重复任务保存一条重复规则，在每个目标周期惰性生成一个独立实例。
- 重复实例 ID 由“规则 ID + 周期键”确定，同一周期重复执行也不会生成两条。
- 修改重复规则不改写已结束周期的历史快照。

### 2.3 状态和周期结算

任务实例状态为：

- `pending`：待办
- `completed`：已完成
- `pass`：周期内未完成

规则：

- 已完成任务在当前周期内保留并划线。
- 日、周、月周期结束后，仍为待办的任务自动变为 Pass。
- Pass 计入未履约，不在下一周期的默认列表中继续展示。
- 历史结果不可改写；需要重做时复制为当前周期的新任务。
- 闲时任务不会自动 Pass，可被重新安排到日、周或月。
- v1 重复任务固定为每个所属周期一次，不支持“每 2 日”等跨周期间隔。
- 周期结算采用同步空间固定时区，首版默认 `Asia/Shanghai`，避免设备时区变化篡改历史。

为降低后台消耗，周期结算采用惰性计算：App、Widget 或同步引擎下次读取数据时补算，不依赖午夜常驻任务。

## 3. 界面与交互

### 3.1 今日视图

macOS 悬浮任务板和 Android Widget 只展示当天的 `day` 任务：

1. 主线
2. 支线
3. 外传

两处均显示完成项划线状态。周、月和闲时任务只在完整 App 中查看。

### 3.2 macOS

- 原生透明悬浮任务板，可移动和调整尺寸。
- 置顶、毛玻璃/透明度、鼠标穿透是三个独立且持久化的设置。
- 快速新增、任务板显隐、置顶和鼠标穿透均支持可自定义全局快捷键；默认分别为 `Shift + Option + 1`、`Shift + Option + 2`、`Shift + Option + 3`、`Shift + Option + 4`。
- 快速新增框中 `Enter` 新增今日主线一次性任务，`Esc` 取消；穿透时强制 20% 不透明度，恢复后回到用户设置。
- 菜单栏始终保留恢复交互的入口，避免穿透后无法操作。
- 面板内支持新增、编辑、完成和组内拖动排序。
- 完整窗口提供周期列表、历史、统计、同步和设置。

首版不实现 WidgetKit 系统小组件。

### 3.3 Android

- 完整 App 提供今日、每日、每周、每月、闲时、历史、统计与设置。
- Samsung 桌面 Widget 支持查看今日任务和直接勾选。
- Widget 的新增、编辑操作打开轻量 Activity。
- 受 Launcher 手势限制，拖动排序在完整 App 中完成。
- 不提供覆盖其他 App 的全局悬浮窗。

### 3.4 自定义纪念日

- 双端顶部默认不显示日期。
- 用户可设置一个标题和起始日期，例如“来到西安 remake”；开启后显示“标题 · 第 N 天”。
- 第一天按起始日期计为第 1 天，未开启时不占用顶部空间。

## 4. 睡前提醒

- 默认每天 23:10 在 Android 本地提醒“列好明日任务再睡觉”。
- 时间可配置，也可关闭。
- 若明日已经存在至少一条任务，当晚不再提醒。
- 每晚最多提醒一次；设备重启后重新安排。
- One UI 省电策略可能造成轻微延迟，首版不申请精确闹钟权限。

### 4.1 任务级提醒

- 有日期的 pending 待办可设置 `HH:mm` 提醒时间，提醒时间随加密任务正文跨端同步。
- macOS 使用 UserNotifications，Android 使用 AlarmManager 与本地通知；两端权限和已显示状态互不复制。
- 完成、Pass、删除、改期、系统时间/时区变化和设备重启后重新计算或取消对应提醒。
- 闲时任务没有日期，不支持任务级提醒。

## 5. 统计

- 履约率：`完成 / (完成 + Pass)`，只计算已结束周期。
- 主线履约率：单独统计主线任务。
- 按每日、每周、每月拆分完成与 Pass。
- 按主线、支线、外传拆分完成与 Pass。
- 展示最近 7 天、8 周、6 个月趋势和样本数。
- 支持按日期进入历史清单。

首版不设计总分、排行榜或跨用户比较，避免通过拆分任务刷分。

## 6. 跨端数据路径

- 默认不配置账号或同步服务，两端各自读写本地 SQLite。
- 推荐使用坚果云 WebDAV 自动同步，每个变更写成不可变 AES-256-GCM 密文对象，不上传 SQLite 或整库 JSON。
- 没有网络时，两端仍可独立编辑本地任务；网络恢复后，已配置的在线同步继续发送积压变更。
- 两端同时修改时，在线同步按 tombstone 永久优先、已结算不退回 pending、截止前完成优先于 Pass 及确定性版本顺序收敛。
- 不使用坚果云时，可自行部署 Worker 并完成配对，启用另一套端到端加密在线增量同步。
- `.wootodo` 只用于用户主动导出的加密恢复备份，不合并到已有任务库。

## 7. 非目标

- 用户名、密码、邮箱登录与多人协作
- iOS、Windows、Web、HarmonyOS
- Android 全局悬浮窗与桌面真实毛玻璃
- 笔记、附件、子任务、标签、自然语言录入
- 秒级实时同步、WebSocket 和云端推送提醒
- 应用商店发布与自动更新
- AI 自律评价、积分和排行榜
