# Woo Todo v0.1.1

这是首个体验修订版，重点补齐睡前规划、macOS 悬浮面板控制、双端配对说明和同步服务的数据安全边界。

## 下载

- `Woo-Todo-v0.1.1-android.apk`：使用项目长期 Release 密钥签名的 Android 安装包。
- `Woo-Todo-v0.1.1-macos-arm64.zip`：Apple Silicon macOS 应用，使用 ad-hoc 签名，未经过 Apple 公证。
- `SHA256SUMS.txt`：两个安装包的 SHA-256 校验值。

Android 签名证书 SHA-256：`77d9b1ff936a9ea9da7ccae4360ede8f1b32b25761378826de7d812bccdba7f7`。

## 主要变化

### macOS

- 增加 20%～100% 日常不透明度预设。
- 开启鼠标穿透后自动使用 20% 不透明度，退出穿透时恢复用户设置。
- 真正区分毛玻璃开关与窗口不透明度；置顶仍由菜单栏独立控制。
- 完成一项任务后，同任务线的剩余待办仍可拖动排序。
- 修复修改重复实例周期后可能被旧规则静默覆盖的问题。
- 全局快捷键冲突时应用继续运行，并提示改用菜单栏操作。
- 补齐与官网一致的 Finder App 图标。

### Android

- 首页增加“明日”清单；23:10 提醒直接打开明日视图，新任务继承当前查看日期。
- 完成 Android 16 edge-to-edge、刘海、IME 与深色系统栏适配。
- 新任务追加到当前周期与任务线末尾；移动到其他组时也按目标组重新排序。
- 屏幕旋转后保留当前清单和未保存的编辑草稿。
- 通知权限被拒绝时明确说明影响，并使用适合状态栏的单色小图标。
- Release 开启 R8 与资源压缩，避免 DEX 方法数逼近上限。
- 数据库降级改为安全拒绝，不再通过删表清空本地任务。

### 配对与同步

- 新建同步空间增加部署者邀请码校验，邀请码只用于首次创建请求，不保存，也不替代设备配对码。
- Mac 创建双端同步空间时明确拒绝 `127.0.0.1/localhost`，并解释它只能指向当前设备。
- 配对页补齐三星系统扫码、六位码确认、首次同步与备用链接流程。
- Android 未连接时提供完整配对步骤，并细分 DNS、超时、连接及证书错误。
- 服务端拒绝同 `opId` 的内容冲突和超前 cursor，避免客户端误删 outbox 或永久跳过远端变更。
- 同步空间创建、活跃设备数与单空间存储均有明确上限，降低免费 D1 被异常请求耗尽的风险。

### 文档与主页

- README、安装指南和 Vercel 主页新增“安装后第一分钟”说明。
- 明确 `Control + Option + Space` 控制鼠标穿透，不是置顶开关。
- 新增完整的双端配对与排错文档。

## 升级前须知

- 升级前建议先导出一份不含同步身份的 `.wootodo` 加密备份。
- Android 正式包沿用 v0.1.0 的签名，可直接覆盖升级；不要使用临时测试签名包覆盖正式安装。
- macOS 包仍未经过 Apple 公证，首次打开新版本可能需要在“系统设置 → 隐私与安全性”中手动允许。
- Android Launcher 的 RemoteViews Widget 不支持应用内 RecyclerView 式拖拽；组件可查看、完成、新增和进入编辑，排序需进入今日清单长按操作。
- 当前没有事务化的“解除本机绑定”入口；不要只删除 Keychain/Keystore 凭据。需要迁移同步空间时先保留加密备份并按配对文档操作。
- Widget、通知时效、窗口层级和长期耗电仍需要在目标真机继续观察；发现明确问题可直接提交 GitHub Issue。
