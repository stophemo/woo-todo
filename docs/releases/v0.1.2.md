# Woo Todo v0.1.2

这一版把“不依赖公网也能跨端接力”落实为默认工作流，并为 macOS 增加类似系统搜索框的全局快速新增入口。在线 Worker 同步继续保留，但只作为用户主动启用的可选增强。

## 下载

- `Woo-Todo-v0.1.2-android.apk`：使用项目长期 Release 密钥签名的 Android 安装包。
- `Woo-Todo-v0.1.2-macos-arm64.zip`：Apple Silicon macOS 应用，使用 ad-hoc 签名，未经过 Apple 公证。
- `SHA256SUMS.txt`：两个安装包的 SHA-256 校验值。

Android 签名证书 SHA-256：`77d9b1ff936a9ea9da7ccae4360ede8f1b32b25761378826de7d812bccdba7f7`。

## 主要变化

### 默认离线接力

- Android 与 macOS 均可导出不含同步身份的加密 `.wootodo` 接力包，并合并到已有任务库。
- 接力包使用 PBKDF2-HMAC-SHA256 与 AES-256-GCM；传输可走 U 盘、局域网文件传输或用户选择的系统分享目标。
- 合并在单个 SQLite 事务内完成；同一文件重复导入不会重复任务或重复写入在线同步 outbox。
- 大小写不同的同一实体 ID 会先规范化并确定性折叠；Wire v1 明确拒绝非 ASCII 标识符，避免跨端数据库大小写规则分歧。
- tombstone 永久阻止同 ID 任务复活；已结算任务不退回 pending；截止前完成优先于 Pass。
- 其余并发修改按 `updatedAt` 与跨端一致的二进制内容指纹确定性收敛。
- 修复任务周期在两端分歧时，完成时间可能超过合并后周期截止点仍被保留的问题。

### Android

- “更多”菜单增加“导出离线接力包”和“导入离线接力包”，与全新安装恢复入口明确分开。
- 接力包既可保存到系统文件选择器，也可直接打开 Android Sharesheet。
- 使用 `FileProvider` 临时共享原子落盘的唯一加密文件，不会因连续分享而让旧 URI 提前失效。
- 已配置可选在线同步时，只有实际合并的变化才进入持久化 outbox。

### macOS

- `Control + Option + N` 从任意应用唤起紧凑新增框；`Enter` 创建今日主线一次性任务，`Esc` 取消。
- 菜单栏新增“快速新增任务”备用入口；原 `Control + Option + Space` 鼠标穿透/交互切换保持不变。
- 快速框在当前屏幕上方居中显示，重复唤起只聚焦现有输入，完成后返回此前使用的应用。
- “任务详情与统计 → 同步”增加离线接力导出与合并，并继续保留单独的恢复备份入口。

### 文档与安全边界

- README、安装、架构和同步安全文档统一为“本地 SQLite + 主动离线接力”的默认路径。
- 明确夸克网盘只能保存用户主动导出的密文文件，不能作为自动实时同步数据库。
- 明确 `127.0.0.1` 只适合单机调试；只有自行部署 Worker 后才需要 HTTPS 地址与二维码配对。

## 升级前须知

- Android 正式包沿用既有签名，可从 v0.1.1 直接覆盖升级；不要用 Debug APK 覆盖正式安装。
- 数据库结构未因离线接力新增迁移，但升级前仍建议先导出一份恢复备份并单独保管口令。
- 离线接力是用户主动触发的文件传递，不会在后台自动发现另一台设备，也不会自动监控网盘文件。
- 接力包拒绝同步身份；只有替换原设备且目标安装为空白时，才使用可选携带同步身份的恢复备份。
- macOS 包仍未经过 Apple 公证，首次打开新版本可能需要在“系统设置 → 隐私与安全性”中手动允许。
- 全局快捷键若被其他应用占用，Woo Todo 会继续运行，可改用菜单栏入口。
- Widget、通知时效、窗口层级和长期耗电仍需在目标真机持续观察；明确问题可提交 GitHub Issue。
