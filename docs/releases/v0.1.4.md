# Woo Todo v0.1.4

这一版修复 Android 今日小组件持续显示“正在加载…”的问题，并简化坚果云双端配置。macOS 四项默认快捷键统一为容易记忆的数字组合；离线跨端接力功能已移除，本地离线编辑与加密恢复备份继续保留。

## 下载

- `Woo-Todo-v0.1.4-android.apk`：使用项目长期 Release 密钥签名的 Android 安装包。
- `Woo-Todo-v0.1.4-macos-arm64.zip`：Apple Silicon macOS 应用，使用 ad-hoc 签名，未经过 Apple 公证。
- `SHA256SUMS.txt`：两个安装包的 SHA-256 校验值。

Android 签名证书 SHA-256：`77d9b1ff936a9ea9da7ccae4360ede8f1b32b25761378826de7d812bccdba7f7`。

## 主要变化

### Android 小组件

- 修正今日任务集合服务的导出声明，桌面宿主可以在系统权限保护下绑定服务。
- 任务行改用明确受支持的 `RemoteViews` API，避免部分桌面因拒绝反射动作而一直停留在加载状态。
- 新增设备 instrumentation 测试，实际应用待办与已完成两种任务行，并校验服务权限声明。
- 数据读取失败会写入不含任务正文的诊断日志，不再静默吞掉异常。

### 坚果云与自建同步

- 推荐先在 Mac 保存坚果云配置，再显式显示 Android 配置二维码；手机扫码后确认并保存，自动生成本机独立设备 ID。
- 二维码只携带严格校验的 `username`、`appPassword`、`vaultId` 和 `vaultKey`，不携带固定端点或设备 ID。
- 修复同步配置输入框的粘贴操作，并让同步状态、配对码、错误提示和详情文本支持选择或复制。
- 文档明确：坚果云客户端用于登录并确认账号，网页端用于生成应用密码，Woo Todo 内置 WebDAV 客户端直接同步。
- 补全 Cloudflare Workers + D1 的部署、迁移、密钥和配对步骤；个人使用可从免费计划开始，实际额度与网络可达性以 Cloudflare 当前政策为准。

### macOS

- 四项默认全局快捷键统一为 `Shift + Option + 1/2/3/4`，依次对应快速新增、任务板显隐、置顶和鼠标穿透。
- 仍使用旧默认值的快捷键会逐项迁移到新默认值；用户自定义过的快捷键保持不变。
- 增加标准编辑菜单，输入框支持系统剪切、复制、粘贴、全选与撤销；详情页只读文本可直接选择复制。

### 备份与本地能力

- 移除手动导出、传递和合并离线跨端接力包的入口与实现。
- `.wootodo` 继续用于加密恢复备份，只能恢复到空白任务库，不会合并已有任务。
- 新增、编辑、完成、删除和排序仍先写本地 SQLite；断网不阻塞使用，已配置的坚果云或 Worker 会在网络恢复后继续同步。

## 升级前须知

- Android 正式包沿用既有签名，可从 v0.1.3 直接覆盖升级；不要用 Debug APK 覆盖正式安装。
- 配置二维码等同坚果云应用密码与 Woo Todo 同步密钥的完整凭据；`wootodo://` 自定义链接无法由 Android 验证应用归属，只能在两台设备近旁使用，并确认扫码结果由 Woo Todo 打开。
- 升级后请启动 Android 应用一次以触发小组件刷新；若桌面仍保留旧缓存，可移除组件后重新添加。
- macOS 包仍未经过 Apple 公证，首次打开新版本可能需要在“系统设置 → 隐私与安全性”中手动允许。
