# Woo Todo v0.1.5

这一版让 Android 可以直接扫描 Mac 生成的同步配置二维码，并重做双端今日顶部显示设置；macOS 与 Android 也会低频检查 GitHub 最新正式版，在有新版时提醒下载。

## 下载

- `Woo-Todo-v0.1.5-android.apk`：使用项目长期 Release 密钥签名的 Android 安装包。
- `Woo-Todo-v0.1.5-macos-arm64.zip`：Apple Silicon macOS 应用，使用 ad-hoc 签名，未经过 Apple 公证。
- `SHA256SUMS.txt`：两个安装包的 SHA-256 校验值。

Android 签名证书 SHA-256：`77d9b1ff936a9ea9da7ccae4360ede8f1b32b25761378826de7d812bccdba7f7`。

## 主要变化

### Android 应用内扫码

- “更多”新增“扫描 Mac 配置二维码”，可直接识别 Mac 同步页生成的坚果云配置二维码。
- 扫码后先显示预填配置，用户确认保存后才会写入本机，并立即执行首次同步。
- Android 使用本机独立的 `deviceId`，不会照搬 Mac 设备身份；取消、无效或被篡改的二维码不会改写现有配置。
- 同一扫码入口继续支持自建 Worker 的一次性配对二维码。
- 相机不是安装应用的必需硬件；无相机或拒绝权限时仍可手动填写配置或使用系统深链入口。

### 自定义今日顶部显示

- macOS 今日面板、Android 今日页和 Android Widget 的标题与副标题均可自由编辑，默认“今日任务”也可以替换或隐藏。
- 模板支持 `{weekday}`、`{elapsedDays}` 和 `{deadlineDays}`，分别显示星期、从起始日计算的耗时天数和距截止日天数。
- 起始日按第 1 天计算，起始日前显示 0；截止日当天显示 0，过期后显示负数。
- 模板变量可从菜单插入，其他文字完全保留；旧版纪念日配置会自动迁移。

### 新版本提醒

- 双端启动或重新进入前台后检查 GitHub 最新正式 Release，成功检查后 24 小时内不重复请求，失败后至少等待 15 分钟再重试。
- 同一新版只自动提醒一次，也可以从应用菜单手动检查。
- 只接受本仓库规范的 `vMAJOR.MINOR.PATCH` 正式标签和下载地址；草稿、预发布及外部地址会被拒绝。
- 应用仅提醒并打开 GitHub 下载页，不会静默下载、自动安装或绕过系统安全检查。

### 发版与兼容性

- Release 工作流增加稳定版本标签门禁，非规范标签会在构建前终止。
- Android Release 继续执行单元测试、Lint、签名构建和独立验签；macOS Release 继续执行全量 Swift 测试、架构检查和签名验证。
- 坚果云二维码 URI 增加 Swift/Kotlin 共用 fixture，覆盖应用密码中的空格、`+`、`/`、`&` 和 `=`。

## 升级前须知

- Android 正式包沿用既有签名，可从 v0.1.4 直接覆盖升级；不要用 Debug APK 覆盖正式安装。
- 首次使用应用内扫码时需要授予相机权限。配置二维码等同坚果云应用密码与 Woo Todo 同步密钥，只能在两台设备近旁展示，用完立即在 Mac 隐藏。
- 自动更新提醒需要能够访问 GitHub；检查失败不会影响本地任务、Widget、通知或已配置的同步。
- macOS 包仍未经过 Apple 公证，首次打开新版本可能需要在“系统设置 → 隐私与安全性”中手动允许。
