# Woo Todo v0.1.6

这一版扩展双端顶部显示模板，支持英文星期和更完整的日期变量；同时完善新版本提醒，让用户可以明确选择更新或忽略当前版本。

## 下载

- `Woo-Todo-v0.1.6-android.apk`：使用项目长期 Release 密钥签名的 Android 安装包。
- `Woo-Todo-v0.1.6-macos-arm64.zip`：Apple Silicon macOS 应用，使用 ad-hoc 签名，未经过 Apple 公证。
- `SHA256SUMS.txt`：两个安装包的 SHA-256 校验值。

Android 签名证书 SHA-256：`77d9b1ff936a9ea9da7ccae4360ede8f1b32b25761378826de7d812bccdba7f7`。

## 主要变化

### 自定义顶部显示

- `{weekdayEn}` 显示英文星期全称，`{weekdayEnShort}` 显示英文星期简称。
- `{weekdayShort}` 提供中文星期简称；原有 `{weekday}`、`{elapsedDays}` 和 `{deadlineDays}` 保持兼容。
- 新增 `{date}`、`{dateLong}`、`{year}`、`{month}`、`{monthPadded}`、`{day}`、`{dayPadded}`、`{startDate}` 和 `{deadlineDate}`。
- Android 与 macOS 的变量插入菜单按星期、日期、计时分组；未知变量和自由文字继续原样保留。

### 新版本提醒

- 发现新正式版时，Android 和 macOS 都提供“更新”和“忽略此版本”。
- 点击“忽略此版本”后，当前版本不再自动提示；之后发布更高版本时仍会提示。
- 点击“更新”会打开 GitHub Release 下载页，不会静默下载、自动安装或绕过系统安全检查。
- 手动检查始终可用，也可以重新查看已经忽略的版本；网络失败不会影响本地任务使用。

## 升级前须知

- Android 正式包沿用既有签名，可从 v0.1.5 直接覆盖升级；不要用 Debug APK 覆盖正式安装。
- macOS 包仍未经过 Apple 公证，首次打开新版本可能需要在“系统设置 → 隐私与安全性”中手动允许。
- 自动更新提醒需要能够访问 GitHub；检查失败不会影响本地任务、Widget、通知或已配置的同步。
