安装后第一分钟

按钮在哪，跨端怎么同步

01

MACOS

从菜单栏控制任务板

  1. 启动后看屏幕右上角菜单栏的清单图标；应用不会出现在 Dock。
  2. 在任何应用中按 ⇧⌥1 唤起快速新增框；Enter 创建今日主线任务，Esc 取消。
  3. ⇧⌥2 显隐任务板；⇧⌥3 切换置顶，菜单内也可开关“毛玻璃”并选择日常不透明度。
  4. ⇧⌥4 切换鼠标穿透；穿透后自动降到 20% 不透明度，再按一次恢复并聚焦。

四项全局快捷键都可在“任务详情与统计 → 快捷键”中自定义；发生冲突时也可从菜单栏执行对应操作。

02

ANDROID

睡前先打开“明日”

  1. 顶部选择“明日”，点右下角 + 连续写下第二天的任务。
  2. 长按三星桌面空白处，选择“组件 → Woo Todo”，添加今日任务组件。
  3. Widget 可直接勾选完成；点任务会进入 App 编辑。

默认每天 23:10 提醒规划；通知权限被关闭时，系统不会显示提醒。

03

JIANGUOYUN SYNC

坚果云自动同步

  1. 先安装并登录坚果云官方客户端确认账号可用，再在网页“账户信息 → 安全选项 → 第三方应用管理”创建应用密码；网页端只用于安全设置。
  2. Mac 打开“任务详情与统计 → 同步 → 坚果云自动同步”，填写账号和应用密码，保存并等待连接成功。
  3. 连接成功后二维码默认隐藏；在 Mac 显式点击“显示 Android 配置二维码”。
  4. Android 在 Woo Todo 中选择“更多 → 扫描 Mac 配置二维码”，确认预填配置并保存；手机会生成本机独立设备 ID。

配置二维码严格包含 username、appPassword、vaultId 与 vaultKey，等同坚果云 WebDAV 与 Woo Todo 的完整敏感凭据。 它不含固定端点或 deviceId，只能在两台设备近旁展示，用完立即隐藏；Woo Todo 不会上传或记录二维码和完整配置链接。任务先写入本地，坚果云只保存 AES-256-GCM 密文。

为每天重复使用而设计

不引入笔记、附件和复杂项目管理。把资源留给任务本身，也把系统资源还给电脑和手机。

A

双端原生，轻量常驻

macOS 使用 AppKit + SwiftUI，Android 使用 Kotlin + RemoteViews。没有 Electron、Flutter，也没有前台常驻服务。

原生
B

本地优先，网络不挡路

新增、修改、完成和排序只依赖本地 SQLite。断网时照常工作，恢复联网后继续自动同步。

离线可用
C

日、周、月与闲时

时间维度与主线、支线、外传交叉使用；重复任务、历史、Pass 和履约趋势都包含在首版中。

清晰分层
D

无传统账号，加密备份

默认不注册账号、不连接服务器。恢复备份使用加密的 .wootodo，可手动保存到自己的网盘或本地介质。

任务明文只留在设备里

两端始终先读写本地 SQLite。启用坚果云后，每次变更会写成 AES-256-GCM 密文对象并自动同步，云端只能看到同步所需的实体标识、逻辑时钟和密文大小等元数据。暂时断网时改动留在本地，恢复联网后继续同步；自建 Worker + D1 也是可选在线方案。

关于跨端同步

推荐使用坚果云 WebDAV 密文自动同步，不需要自建服务器、代理或额外服务器费用；也可自行部署 Cloudflare Workers + D1 作为可选在线方案。自建路径无需购买域名，按当前免费计划的额度可先服务个人双端，但 Cloudflare 的额度、计费与地区网络政策会变化。断网期间两端仍可独立使用，恢复联网后再同步积压变更。
